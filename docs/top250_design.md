# Кинопоиск Топ 250 — дизайн ветки

## Роль в флоу

Отдельная ветка после развилки «Подобрать фильм»: пользователь выбирает **Кинопоиск Топ 250**, далее идёт по шагам только этой ветки (не смешивается с обычным подбором по ИИ).

---

## Шаги (последовательно)

1. **Настроение / тип просмотра**  
   Фильтр по тому, как пользователь хочет смотреть (лёгкий вечер, глубокое кино, с семьёй и т.п.).  
   *Уточнить:* тот же набор кнопок, что в обычном подборе, или свой сокращённый набор под «тип просмотра».

2. **Жанр**  
   Один или несколько жанров — сужаем топ по жанру.

3. **Год (эпоха)**  
   Кнопки:
   - **Новое (2010+)**
   - **90-е–00-е** (1990–2009)
   - **Классика (до 1990)**
   - **Не важно**

После трёх шагов — выдача нескольких фильмов из топа по выбранным фильтрам.  
*Уточнить:* сколько фильмов показывать (например 5, как в обычном подборе). - да 5, как в обычном подборе

---

## Источник данных

- **Фильмы топа** — из API Кинопоиска (poiskkino.dev). Топ собирается по сортировке (голоса/рейтинг), 5 страниц по 50 = до 250 фильмов. **Один цикл обновления = 5 запросов к API.** При лимите 200 запросов/день этого хватает с запасом.
- **Частота обновления:** раз в **месяц** (1-е число в 04:00), не каждый день.
- **Схема работы:** раз в месяц забираем Топ 250 с API и записываем в таблицу **kinopoisk_top250**. Бот при подборе читает только из БД (внешний API не дергаем).
- **Поля таблицы:** kinopoisk_id, title, year, genres, rating_kp, position, age_rating, **poster_url**, updated_at. Постер при наличии отправляется в карточке фильма в Telegram (фото + подпись).
