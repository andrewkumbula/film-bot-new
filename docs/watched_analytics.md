# Аналитика: «Смотрел» — переименование, таблица, исключение из рекомендаций

## Требования

1. **Переименовать кнопку** «Посмотрел» → **«Смотрел»** (и вариант «✅ Посмотрел» → «✅ Смотрел»).
2. **Не предлагать** пользователю фильмы, которые он уже отметил как «Смотрел».
3. **Отдельная таблица** для хранения всех «смотрел» по пользователям.

---

## Текущее состояние

### Таблица `watched`

Таблица **уже есть** и используется по назначению:

- **Схема:** `id`, `user_id`, `movie_id`, `created_at`; `UNIQUE(user_id, movie_id)`; `FOREIGN KEY (movie_id) REFERENCES movies(id)`.
- **Индекс:** `idx_watched_user` по `user_id`.
- **Сервис** (`app/services/favorites.py`):
  - `is_watched(user_id, rec)` — проверка, есть ли фильм в «смотрел» у пользователя (по `movie_id` через kinopoisk_id / title+year).
  - `add_watched_for_user(user_id, rec)` — добавление записи в `watched` (с созданием/поиском `movie_id` в `movies`).

Итого: отдельная таблица под «смотрел» уже реализована, менять схему не требуется.

### Текст кнопки

Сейчас везде используется:

- До нажатия: **«🎬 Посмотрел»**
- После нажатия: **«✅ Посмотрел»**

Нужно заменить на **«Смотрел»** / **«✅ Смотрел»** во всех местах (обычный подбор и Топ 250).

### Исключение «смотрел» из рекомендаций

Сейчас **не делается**:

- **Обычный подбор** (`_do_recommendations`): отфильтровываются только фильмы с рейтингом Кинопоиска &lt; 6.0 и фильмы из «Не интересно». Фильмы из «Смотрел» **продолжают попадать** в выдачу.
- **Топ 250:** кандидаты фильтруются только по «Не интересно» (`ni_kp`). Фильмы из «Смотрел» **тоже не исключаются**.

То есть требование «то, что смотрел, не предлагать» сейчас не выполняется.

## Что нужно сделать (без кода — только план)

### 1. Переименование кнопки

- Заменить все вхождения текста кнопки **«Посмотрел»** и **«✅ Посмотрел»** на **«Смотрел»** и **«✅ Смотрел»** в:
  - обычном подборе (карточка рекомендации, обработчики fav/watched/not_interested);
  - подборе Топ 250 (карточка, t250_fav, t250_watched, t250_not_interested).
- Документация/комментарии: при необходимости обновить формулировки «список „Посмотрел“» → «список „Смотрел“».

### 2. Исключение «смотрел» из рекомендаций

**Обычный подбор (`_do_recommendations`):**

- После фильтра по рейтингу и до/после фильтра «Не интересно» добавить шаг: по текущему `user_id` получить множество `movie_id` (или kinopoisk_id) из таблицы `watched`.
- Из `filtered_pairs` убрать пары, у которых фильм входит в это множество (по `movie_id` или по совпадению с записью в `watched` через kinopoisk_id/title+year).
- Обработать сценарий «все рекомендации оказались в „Смотрел“»: сообщение вроде «Все эти варианты ты уже смотрел. Нажми «Подобрать ещё» — подберу другие» и не показывать пустой список.

**Топ 250 (`top250_year_era`):**

- После получения `candidates` из `get_filtered_top250` и исключения «Не интересно» дополнительно исключить фильмы, которые есть у пользователя в `watched` (по `kinopoisk_id` или `movie_id`).
- Для этого нужен сервисный метод вида `get_watched_kinopoisk_ids(user_id)` и/или `get_watched_movie_ids(user_id)` (аналогично `get_not_interested_kinopoisk_ids`), чтобы отфильтровать список кандидатов до передачи в ИИ/рандома.
- При пустом списке после фильтра — сообщение вроде «По твоим фильтрам всё из Топ 250 ты уже смотрел» и не показывать карточки.

### 3. Таблица и сервис

- Таблицу **`watched`** не переименовывать и не менять: она уже хранит все «смотрел» по пользователям.
- При необходимости вынести получение списка «смотрел» в отдельные функции (например, `get_watched_movie_ids`, `get_watched_kinopoisk_ids`) в `favorites.py` или отдельном модуле, чтобы и обычный подбор, и Топ 250 использовали один и тот же способ исключения.

---

## Риски и краевые случаи

- **Один и тот же фильм в разных источниках:** идентификация по `movie_id` (и при необходимости по kinopoisk_id / title+year) уже реализована в `is_watched` и `add_watched_for_user` — важно использовать ту же логику при фильтрации (чтобы не показывать «смотрел» в одной ветке и показывать в другой).
- **Пустая выдача:** если после исключения «смотрел» и «не интересно» не остаётся ни одной рекомендации, пользователю нужно явное сообщение и кнопка «Подобрать ещё», без пустого списка.
- **Производительность:** при росте таблицы `watched` запрос по `user_id` уже закрыт индексом `idx_watched_user`; при очень больших объёмах можно позже добавить кэш по user_id на время одного запроса подбора.

---

## Итог

- Таблица под «смотрел» уже есть (`watched`), её менять не нужно.
- Нужно: (1) переименовать кнопку «Посмотрел» → «Смотрел» во всём UI; (2) в обычном подборе и в Топ 250 исключать из рекомендаций фильмы из `watched` для текущего пользователя и обрабатывать случай «все отфильтровались».

После утверждения этой аналитики можно переходить к разработке (правки в `flow_movie.py`, при необходимости в `favorites.py` и в `top250`).
