# Аналитика: кнопка «Не интересно» на карточке фильма

## Цель

- Добавить на карточку фильма кнопку **«Не интересно»**.
- Сохранять такие фильмы в отдельной таблице в связке с пользователем.
- Не показывать пользователю фильмы из этой таблицы в последующих подборах.
- После нажатия кнопка меняет вид (визуальное подтверждение).

---

## 1. Хранение данных

### Новая таблица

- **Имя (варианты):** `not_interested` / `user_not_interested`.
- **Назначение:** связка «пользователь — фильм, который он отметил как неинтересный».

**Поля (по аналогии с `favorites` и `watched`):**

| Поле       | Тип    | Описание                          |
|-----------|--------|-----------------------------------|
| id        | INTEGER| PRIMARY KEY AUTOINCREMENT         |
| user_id   | INTEGER| NOT NULL, Telegram user id       |
| movie_id  | INTEGER| NOT NULL, FK → movies(id)        |
| created_at| TIMESTAMP | время добавления               |

- **UNIQUE(user_id, movie_id)** — один и тот же фильм пользователь может отметить один раз.
- **Индекс** по `user_id` для быстрой выборки «все неинтересные для пользователя».

**Связь с фильмом:** через `movie_id` (как в избранном и «Посмотрел»). Если в момент нажатия фильма ещё нет в `movies` — его нужно сначала создать/получить (как при добавлении в избранное), затем записать в `not_interested`.

---

## 2. Где показывается карточка фильма

Карточка с кнопками есть в двух потоках:

1. **Обычный подбор** (`MovieFlow.recommendations`) — рекомендации после опроса (настроение, жанр и т.д.), данные от LLM + Кинопоиск.
2. **Кинопоиск Топ 250** (`Top250Flow.recommendations`) — подборка из Топ 250 после выбора настроения/жанра/эпохи.

В обоих местах нужно:
- добавить кнопку «Не интересно» на карточку;
- при формировании следующих рекомендаций исключать фильмы из `not_interested` для данного пользователя.

---

## 3. Фильтрация: где не показывать «неинтересные» фильмы

- **Обычный подбор:** после получения ответа LLM (список из 5 фильмов) отфильтровать те, у которых `movie_id` (или пара title+year/kinopoisk_id) есть в `not_interested` для текущего `user_id`. Если после фильтрации остаётся меньше 5 — можно оставить как есть (показать меньше) или не фильтровать по «не интересно» на этапе LLM (фильтровать только перед показом, подставляя следующие из кандидатов — сложнее). Простой вариант: фильтровать по `not_interested` после LLM, показывать только «не отмеченные»; если все 5 отмечены — показать сообщение «Все эти варианты ты уже отметил как неинтересные, нажми „Подобрать ещё“».
- **Топ 250:** при формировании списка кандидатов (или после выбора ИИ) исключать фильмы, чей `movie_id` (или kinopoisk_id) есть в `not_interested` у пользователя. То есть `get_filtered_top250` или слой выше должен получать `user_id` и вычитать из результата записи из `not_interested`.

Важно: фильтрация должна быть по `user_id` — у каждого пользователя свой список «не интересно».

---

## 4. Кнопка: вид и поведение

- **До нажатия:** текст вида «👎 Не интересно» (или «Не интересно» без эмодзи).
- **После нажатия:** вид меняется, чтобы было понятно, что действие учтено, например:
  - «✅ Не интересно»;
  - или «✓ Не интересно»;
  - или оставить тот же текст, но визуально неактивная кнопка (в Telegram InlineKeyboard «неактивность» можно передать только сменой текста/эмодзи).

Рекомендация: сменить текст на «✅ Не интересно» и при следующем открытии списка избранного/истории не использовать; при повторном показе карточки (например, при «Подобрать ещё») эта карточка пользователю уже не показывается.

**Callback:** по аналогии с `fav:` и `watched:`:
- обычный подбор: `not_interested:{idx}`;
- Топ 250: `t250_not_interested:{idx}`.

По `idx` берётся запись из текущего списка рекомендаций в state, из неё — идентификатор фильма (kinopoisk_id / title+year) для получения/создания `movie_id` и записи в `not_interested`.

---

## 5. Сценарии

| Действие | Что сделать |
|----------|-------------|
| Пользователь нажимает «Не интересно» | Записать в `not_interested` (user_id, movie_id). Обновить сообщение: кнопка меняет вид на «✅ Не интересно». |
| Следующий подбор (обычный или Топ 250) | При формировании списка рекомендаций исключать фильмы, у которых (user_id, movie_id) есть в `not_interested`. |
| Фильм ещё не в `movies` | Как при «В избранное»: при необходимости получить/создать запись в `movies` (по kinopoisk_id или title+year), затем записать в `not_interested` по `movie_id`. |

---

## 6. Отмена «Не интересно» (опционально)

В первой версии можно не делать. Если потом понадобится:
- добавить снятие отметки (удаление строки из `not_interested`);
- при снятии кнопка снова могла бы показывать «👎 Не интересно» (если бы мы снова показывали эту карточку); логически это потребует либо отдельной сущности «история неинтересных» с возможностью отмены, либо просто удаления из `not_interested` без смены кнопки на уже отправленной карточке.
