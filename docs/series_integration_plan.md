# План интеграции подбора сериалов в бота «Фильм на вечер»

## Исходные данные

- **Текущий бот:** подбор только фильмов. Главное меню: «Подобрать фильм» → источник (Обычный / Топ 250 / Оскар) → анкета → выдача.
- **ТЗ (telegram_series_bot_spec.md):** подбор сериалов: время → формат → настроение → ограничения → 3 карточки; отдельные флоу «По настроению», «Похоже на сериал», «Случайный», «Популярные»; избранное и «Уже смотрел».

Цель: один бот для фильмов и сериалов с понятным входом и без дублирования логики там, где можно переиспользовать.

---

## 1. User flow: главный экран и первый шаг

### Вариант A — Раздельные кнопки на главном экране (рекомендуется)

**Главный экран:**
```
Что смотрим?
[🎬 Подобрать фильм]   [📺 Подобрать сериал]
[⭐️ Избранное]        [⚙️ Настройки]
[ℹ️ Помощь]
```

- Один тап — сразу в нужный тип контента.
- «Подобрать фильм» — текущий флоу без изменений (источник → анкета → выдача).
- «Подобрать сериал» — новый флоу по ТЗ (время → формат → настроение → ограничения → 3 сериала).
- Избранное: общий список или две вкладки «Фильмы» / «Сериалы» (см. ниже).

**Плюсы:** быстро, привычно, не перегружаем один сценарий.  
**Минусы:** две кнопки вместо одной.

---

### Вариант B — Один вход, первый шаг «Фильм или сериал»

**Главный экран:**
```
[🎬 Подобрать фильм или сериал]
[⭐️ Избранное]  [⚙️ Настройки]  [ℹ️ Помощь]
```

После нажатия — инлайн-кнопки:
```
Что подобрать?
[🎬 Фильм]  [📺 Сериал]
```

Дальше: Фильм → текущий source_keyboard; Сериал → флоу сериалов.

**Плюсы:** один главный сценарий, явный выбор типа.  
**Минусы:** лишний шаг для тех, кто всегда выбирает только фильмы или только сериалы.

---

### Рекомендация: **Вариант A**

- Соответствует формулировке ТЗ «на главном экране… подобрать сериал или фильм».
- Меньше шагов для постоянных пользователей.
- Проще объяснять: «для фильма — одна кнопка, для сериала — другая».

**Итог по главному экрану:**
- Добавить кнопку «📺 Подобрать сериал» рядом с «🎬 Подобрать фильм».
- Текст приветствия на /start слегка обновить: «подобрать фильм или сериал».
- Остальные пункты меню (Избранное, Настройки, Помощь) общие.

---

## 2. Избранное и «Уже смотрел»

- **Избранное:** один раздел «Избранное» с двумя подразделами (табы или кнопки): «Фильмы» и «Сериалы». Либо один общий список с пометкой типа (иконка 🎬/📺). Рекомендация: два подраздела — меньше путаницы, проще реализация (отдельные таблицы/флаги).
- **Уже смотрел:** отдельные списки для фильмов (как сейчас) и для сериалов; при подборе сериалов исключать только сериалы из «уже смотрел», при подборе фильмов — только фильмы.
- **Настройки:** общие (один пользователь — одни настройки); при необходимости добавить опции только для сериалов (например, «рейтинг 7+» по умолчанию).

---

## 3. Структура данных сериалов

Имеет смысл завести отдельную сущность «сериал», не смешивая с фильмами в одной таблице (разные поля: сезоны, серии, статус, формат).

**Предлагаемая схема:**

- **Таблица `series`** (по аналогии с `movies`):
  - id, kinopoisk_id (или kp_id), name, original_name, year
  - rating, votes, poster_url, poster_urls, description (short_description)
  - is_mini_series, seasons_total, episodes_total, runtime_episode_min
  - status (ended / ongoing), countries, genres
  - опционально: binge_score, lightness_score, intensity_score (если будут в API или расчёте)
  - created_at, updated_at

- **Таблицы по пользователям:**
  - `series_favorites` (user_id, series_id) — избранные сериалы
  - `series_watched` (user_id, series_id) — уже смотрел

- **Кэш/источники:** при необходимости отдельная таблица под «популярные подборки» или теги (топ мини-сериалов и т.д.), либо теги в `series`.

Справочники настроения/формата/ограничений можно хранить в коде (enum/константы) или в отдельной маленькой таблице — на старте достаточно кода.

## 4. Источник данных по сериалам

- Использовать тот же **Kinopoisk API** (poiskkino.dev / неофициальный API), если есть поиск/фильтрация по сериалам и поля: сезоны, серии, длительность серии, статус.
- Проверить по документации API: методы для сериалов, поиск по жанру/рейтингу, лимиты.
- Если готовых «топов» для сериалов нет — на первом этапе можно формировать подборку из поиска по жанру/настроению + фильтрация по длительности/формату на нашей стороне.

## 5. Этапы разработки и встраивания в текущий флоу

### Фаза 0 — Подготовка (без смены UX)

- Уточнить и зафиксировать выбранный user flow (здесь: Вариант A).
- Проверить Kinopoisk API на предмет сериалов (эндпоинты, поля, лимиты).
- Добавить таблицы `series`, `series_favorites`, `series_watched` и миграции.
- Реализовать минимальный сервис доступа к сериалам (сохранение в БД, получение по id, по фильтрам для подбора).

### Фаза 1 — Главный экран и MVP флоу «Подобрать сериал на вечер»

- **Главное меню:** добавить кнопку «📺 Подобрать сериал»; обновить текст приветствия и, при необходимости, Помощь.
- **Роутер/FSM для сериалов:** новый `SeriesFlow` (или несколько групп состояний по сценариям).
- **Сценарий по ТЗ «Подобрать сериал на вечер»:**
  1. Шаг 1 — Сколько времени: 1–2 ч / 2–4 ч / Несколько вечеров / Не важно.
  2. Шаг 2 — Формат: Мини-сериал / 1 сезон / Несколько сезонов / Не важно.
  3. Шаг 3 — Настроение: Лёгкое, Напряжённое, Смешное, Атмосферное, Мрачное, Романтика, Удиви.
  4. Шаг 4 — Ограничения (мультивыбор): Только завершённые, Без ужасов, Без тяжёлой драмы, Рейтинг 7+, Без российских.
  5. Шаг 5 — Выдача: 3 сериала, карточки по формату из ТЗ (название, год, жанры, рейтинг, описание, серии/длительность/сезоны, «Почему подходит»). Кнопки: Смотреть, Похожее, Другой вариант, В избранное, Уже смотрел.
- Фильтрация: исключать сериалы из `series_watched`, применять рейтинг и остальные ограничения.
- Интеграция с API: получение списка кандидатов (по жанру/настроению/длительности), сохранение в `series`, отображение карточек.

### Фаза 2 — Дополнительные флоу сериалов

- «Подбор по настроению»: выбор настроения → темп (быстро затягивает / спокойный / баланс) → 3 сериала.
- «Похоже на сериал»: ввод названия → что понравилось (Сюжет, Атмосфера, Юмор, Персонажи, Напряжение) → выдача похожих.
- «Случайный сериал»: один сериал + кнопки Беру / Другой / В избранное.
- «Популярные подборки»: Топ сериалы, Мини-сериалы, Лучшие детективы, Лучшие комедии, Самые напряжённые → список карточек.

Реализация по приоритету после стабилизации MVP.

### Фаза 3 — Избранное и «Уже смотрел»

- Избранное: разделы «Фильмы» и «Сериалы» (или общий список с типом).
- «Уже смотрел» для сериалов: экран списка + исключение из подборов.
- Обновить текст Помощи под фильмы + сериалы.

### Фаза 4 — Улучшения (по ТЗ п.15)

- Персональный рейтинг, подбор по актёру/режиссёру/платформе, рекомендации по истории и лайкам — после закрепления основных сценариев.

## 6. Технические моменты

- **Роутеры:** вынести флоу сериалов в отдельный модуль (например, `routers/flow_series.py`), подключать в `app/bot.py` рядом с `flow_movie`.
- **Клавиатуры:** общие хелперы в `keyboards/flow.py` или отдельный `keyboards/series.py` для кнопок сериалов (время, формат, настроение, ограничения).
- **Состояния:** отдельная группа состояний для сериалов (например, `SeriesMainFlow`: time, format, mood, restrictions, recommendations), при необходимости отдельные для «по настроению», «похоже на», «случайный», «популярные».
- **Логика подбора:** сервис `services/series.py` (или `series_recommendation.py`) — фильтрация по времени/формату/настроению/ограничениям, выбор 3 кандидатов (из БД/API), учёт «уже смотрел» и рейтинга.
- **ИИ:** при необходимости «Почему подходит» и похожие сериалы — вызов той же OpenRouter (отдельный промпт под сериалы), по аналогии с фильмами.
