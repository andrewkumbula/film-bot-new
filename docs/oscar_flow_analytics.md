# Оскар: аналитика парсинга, хранения и флоу подбора

Только аналитика, без кода. Источник: [Премия «Оскар» за лучший фильм (Википедия)](https://ru.wikipedia.org/wiki/Премия_«Оскар»_за_лучший_фильм).

---

## 1. Парсинг и загрузка данных

### 1.1 Структура страницы Википедии

- **Блок «Победители и номинанты»** — подзаголовки по десятилетиям: `### 1929—1930`, `### 1931—1940`, … до последних лет.
- В каждом блоке — **таблица** с колонками:
  - **Год/Церемония** — например `[1929](...)`, `[1930-I](...)`, `[1930-II](...)`, `[2025](...)`, `[2026](...)`.
  - **Фильм** — название в кавычках «…», часто со ссылкой. Победитель помечен символом **★** перед названием.
  - Кинокомпания(и), Продюсер(ы), Ссылки — для нас опциональны (можно сохранять для отображения «интересных моментов» или позже).

**Что извлекать обязательно:**

| Поле | Описание | Примеры |
|------|----------|---------|
| Церемония / год | Год церемонии (нормализованный) | 1929, 1930, 2024, 2025 |
| Метка церемонии | Как в Вики (для уникальности) | 1929, 1930-I, 1930-II |
| Название фильма | Текст без разметки, из ячейки «Фильм» | Крылья, Унесённые ветром, Оппенгеймер |
| Победитель / номинант | Флаг: есть ★ → победитель | true / false |

**Особенности:**

- В 1930 году две церемонии (I и II) — две строки «года», различать по метке.
- Год в ссылке/тексте может быть разным от года выхода фильма (церемония — за предыдущий год). Для подбора пользователя логично опираться на **год церемонии** (или хранить оба: год церемонии + при необходимости год фильма из ссылки).
- Названия — в кавычках «…», внутри могут быть ссылки `[Текст](url)` — нужна очистка до чистого названия.

### 1.2 Другие номинации (на будущее)

- Сейчас парсим только **«Лучший фильм»** (одна страница).
- Другие номинации — отдельные страницы Вики (лучший режиссёр, лучший актёр и т.д.). Имеет смысл заложить в модель данных **тип номинации** (категорию), чтобы потом подключать парсеры других страниц без смены схемы.
- Предлагаемое значение категории для текущей страницы: например `best_picture` (или `oscar_best_picture`).

### 1.3 «Интересные моменты»

- На странице есть блок **«Курьёзы при вручении»**: короткие истории (1985 — Оливье объявил победителя сразу; 2017 — Ла-Ла Ленд/Лунный свет; 2021 — порядок объявления и т.д.).
- Ниже — **«Достижения фильмов»**: таблицы по другим номинациям, рекорды (первый фильм с «большой пятёркой», последний номинант без других номинаций и т.д.).

**Хранение:** в таблице Оскара (отдельная таблица `oscar_moments`): год (или год+категория), тип (курьёз / достижение), текст/краткое описание. Связь с фильмами по году и при необходимости по названию. Для первого этапа достаточно парсить и сохранять курьёзы по году церемонии; при показе карточки в разделе Оскара можно подмешивать текст для этого года (если есть).

## 2. Модель данных (БД)

### 2.0 Принцип размещения данных

- **Все фильмы попадают в основную таблицу `movies`** — название, год, постер, рейтинг, описание, жанры и т.д. Единственный источник правды о фильме как о сущности.
- **В таблице Оскара хранятся только данные об Оскаре** — какие фильмы номинированы/победили, в какой категории и в каком году, а также **интересные моменты** (курьёзы, достижения по годам). Никакого дублирования полей фильма (название, постер, рейтинг и т.п.) в таблице Оскара нет: только связь с `movies` через `movie_id`, служебные поля номинации и данные об интересных моментах.
- Итог: фильм один раз лежит в `movies`; запись в таблице Оскара лишь говорит «этот фильм (movie_id) — номинант/победитель в категории X за год Y». При показе карточки данные всегда берутся из `movies` по `movie_id`.

(Как в архитектуре Топ 250: `movies` — все фильмы; отдельная таблица хранит только признак и позицию/метаданные списка.)

### 2.1 Таблица связи с Оскаром

Таблица **только для номинаций Оскара** — в ней только фильмы Оскара (номинанты/победители), без дублирования данных о фильме:

- **`oscar_nominations`** (или `oscar_entries`):
  - `id` (PK).
  - `category` — категория номинации: `best_picture`, позже `best_director`, `best_actor` и т.д.
  - `ceremony_year` — год церемонии (целое число: 1929, 1930, …).
  - `ceremony_label` — исходная метка из Вики (например `1930-II`) для однозначности.
  - `title_from_source` — название из Вики (только для маппинга и отображения до привязки к `movies`).
  - `is_winner` — булев флаг: победитель в этой категории в этом году.
  - `movie_id` (FK → `movies.id`, nullable) — ссылка на запись фильма в основной таблице; после маппинга по Кинопоиску сюда пишем id из `movies`.
  - Опционально: `year_from_source` (год фильма из ссылки Вики), `raw_company`, `raw_producers`.

Поля фильма (название, постер, рейтинг и т.д.) в этой таблице **не хранятся** — они всегда берутся из `movies` по `movie_id`.

Уникальность: одна запись = одна номинация в одной категории в одном году (например, один фильм — одна строка на «Лучший фильм» за 2024 год).

### 2.2 Откуда берётся фильм (`movie_id`)

- При первом парсинге Вики в `oscar_nominations` пишем только данные номинации; фильм как сущность ещё не создан, `movie_id` = NULL.
- Маппинг: по `title_from_source` (+ год) ищем фильм в Кинопоиске; **сначала** создаём или обновляем запись в **`movies`** (туда попадают все данные о фильме — постер, рейтинг, описание и т.д.), **затем** в `oscar_nominations` проставляем `movie_id` на эту запись. В таблице Оскара остаются только номинационные поля.
- Если маппинг не удался — запись Оскара остаётся с `movie_id = NULL`; в подборе можно либо не показывать такие позиции, либо показывать только текст «Номинант/Победитель 20XX: Название» без карточки из `movies`.

### 2.3 Интересные моменты (в таблице Оскара)

- Интересные моменты храним **в части схемы Оскара**, не в `movies`. Отдельная таблица, например **`oscar_moments`**: `id`, `ceremony_year`, `category` (nullable), `kind` (например `curiosity` / `achievement`), `title`, `description` (текст). При желании можно связать с `movie_id` или оставить только привязку по году. Используются при показе подборки по Оскару (например курьёз года в карточке или отдельным блоком).

### 2.4 Атрибуты Оскара в таблице `movies`

- В **основной таблице фильмов** добавляем два атрибута, чтобы показывать статус Оскара в **любом подборе** (обычный подбор, Топ 250, избранное и т.д.), не только в разделе «Оскар»:
  - **`nominated_oscar`** (boolean) — фильм хотя бы раз номинировался на Оскар (например в «Лучший фильм» или в других категориях в будущем).
  - **`won_oscar`** (boolean) — фильм хотя бы раз побеждал на Оскаре.
- Значения выставляются при маппинге: после проставления `movie_id` в `oscar_nominations` обновляем у соответствующей записи в `movies` флаги (если есть хотя бы одна номинация — `nominated_oscar = 1`; если есть хотя бы одна победа — `won_oscar = 1`). При добавлении новых номинаций/побед — обновлять флаги так же.
- В карточке фильма (во всех флоу) можно выводить, например: «🏆 Победитель Оскара» или «📋 Номинант Оскара» — по этим полям из `movies`, без обращения к таблице Оскара.

## 3. Интеграция с Кинопоиском

- После парсинга Вики имеем: название (рус.) + год церемонии (и при возможности год фильма).
- Алгоритм как в обычном подборе / Топ 250: поиск по названию, допуск по году ±1, сохранение в `movies`, проставление `movie_id` в `oscar_nominations`.
- Запуск маппинга:
  - **Вариант A:** фоново после каждого парсинга (по списку записей с `movie_id IS NULL`).
  - **Вариант B:** по требованию при первом заходе пользователя в раздел «Оскар» или при первом показе подборки за выбранный период.
- Лимиты API Кинопоиска те же: не перегружать запросами; при большом объёме — батчи и задержки.

## 4. Флоу подбора в боте (Оскар)

Пользователь уже заходит в ветку «Оскар (номинанты и победители)». Предлагаемые шаги:

### 4.1 Шаг 1: Тип — победитель / номинант / все

- **Только победители** — фильмы с `is_winner = true` в выбранной категории.
- **Только номинанты** — только без победы (но в shortlist того же года).
- **Все** — и победители, и номинанты.

Кнопки, например: «🏆 Только победители», «📋 Только номинанты», «Всё подряд».

### 4.2 Шаг 2: Год (крупные мазки)

Чем старее период, тем крупнее диапазоны, чтобы не раздувать список и не дробить выбор:

- **Новое (2020+)** — 2020, 2021, 2022, 2023, 2024, 2025.
- **2010–2019**
- **2000–2009**
- **1990–1999**
- **1980–1989**
- **1970–1979**
- **1960–1969**
- **Классика (до 1960)** — одна кнопка на всё до 1960 года.
- **Не важно** — все годы (для «покажи любые 5»).

При необходимости для самых старых можно объединить ещё сильнее (например «до 1970» одной кнопкой). Имеет смысл проверять по реальному количеству записей в БД: если в «до 1960» сотни фильмов — разбивка по десятилетиям может быть избыточной.

### 4.3 Дополнительные этапы (опционально)

- **Жанр** — если у нас уже есть маппинг на `movies` и в них подтянуты жанры с Кинопоиска, можно фильтровать по жанру (как в Топ 250). Иначе этот шаг пропускаем или делаем позже.
- **Количество** — фиксировать 5 карточек (как в остальных флоу) или дать выбор «3 / 5 / 10».
- **Случайная выдача** — из отфильтрованного набора (победитель/номинант + период) выбираем случайные N фильмов; при наличии маппинга показываем карточки из `movies` (постер, рейтинг, краткое описание), иначе — текст «Оскар 20XX: Название» и при желании курьёз года.

### 4.4 Учёт «смотрел» и «не интересно»

- Как в Топ 250 и обычном подборе: фильмы из `user_watched` и `not_interested` по `user_id` исключаем из выдачи.

### 4.5 Минимальный рейтинг

- Если включена настройка «не показывать фильмы с рейтингом ниже 6.0», применять её и к карточкам Оскара (по данным из `movies`). Для записей без `movie_id` рейтинга нет — правило можно не применять или считать, что они проходят.

### 4.6 Отображение статуса Оскара во всех подборах

- В карточке фильма в **любом** подборе (обычный подбор, Топ 250, избранное и т.д.) при наличии в `movies` полей `won_oscar` / `nominated_oscar` выводить пометку: например «🏆 Победитель Оскара» или «📋 Номинант Оскара». Данные брать из `movies`, без JOIN с таблицей Оскара — так карточка остаётся быстрой и единообразной во всех флоу.

## 5. Этапы реализации (логический порядок)

1. **Парсер Вики** — скачивание/парсинг страницы «Оскар за лучший фильм», извлечение таблиц по десятилетиям, нормализация года и названия, флаг победитель/номинант. Сохранение в таблицу с `movie_id = NULL`.
2. **Миграции БД** — создание `oscar_nominations`, при необходимости `oscar_moments`; в `movies` добавить колонки `nominated_oscar`, `won_oscar` (boolean).
3. **Маппинг на Кинопоиск** — скрипт/сервис: по записям с `movie_id IS NULL` поиск в Кинопоиске, создание/обновление `movies`, обновление `movie_id` и проставление в `movies` флагов `nominated_oscar` / `won_oscar`.
4. **Парсер «интересных моментов»** — опционально: блок «Курьёзы при вручении» и при желании «Достижения» по годам, сохранение в `oscar_moments`.
5. **Флоу в боте** — шаг 1 (победитель/номинант/все), шаг 2 (год/период), выборка из БД, выдача 5 карточек (из `movies` по `movie_id` или текстовая заглушка), учёт «смотрел»/«не интересно» и минимального рейтинга.
6. **Расширение на другие номинации** — при появлении парсеров других страниц Вики те же таблицы, новая `category`.

## 6. Риски и открытые моменты

| Вопрос | Варианты / примечание |
|--------|------------------------|
| Стабильность разметки Вики | Разметка может меняться; парсер лучше тестировать на снимке HTML/статьи и предусмотреть повторный прогон и обновление записей (upsert по ceremony_year + category + title). |
| Дубли названий в одном году | В одном году все названия в shortlist уникальны; уникальность по (category, ceremony_year, title_from_source) или по (category, ceremony_label, title_from_source). |
| Год фильма vs год церемонии | Оскар вручают за фильмы предыдущего года. Для отображения и Кинопоиска чаще нужен год выхода фильма; при парсинге из ссылки на статью о фильме можно вытащить год и хранить в `year_from_source`. Иначе — ориентироваться на год церемонии и при поиске в Кинопоиске допускать ±1–2 года. |
| Частота обновления парсера | Раз в год после церемонии достаточно; можно запускать вручную или по крону раз в год (например март). |

Если нужно, следующий шаг — зафиксировать схему таблиц (SQL) и формат парсера (вход/выход) в отдельном файле или в этом же документе.
