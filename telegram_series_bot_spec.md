# Техническое задание
## Telegram‑бот для подбора сериалов на вечер

## 1. Цель проекта
Создать Telegram‑бота, который помогает пользователю быстро подобрать сериал для просмотра. Основная задача — сократить время выбора и предложить релевантные варианты на основе настроения, времени и предпочтений пользователя.

Бот должен работать максимально просто:
- минимум ввода текста
- максимум кнопок
- быстрый подбор за 3–4 шага


---

# 2. Основной UX принцип

Пользователь чаще всего не знает точный жанр сериала.

Он думает в категориях:

- "хочу что‑то лёгкое"
- "чтобы затянуло"
- "на один вечер"
- "что‑то короткое"
- "что‑то смешное"

Поэтому подбор строится через:

1. сценарий просмотра
2. настроение
3. формат сериала


---

# 3. Главный экран бота

При запуске бот показывает меню:

```
Что хотите посмотреть?
```

Кнопки:

- 🎯 Подобрать сериал на вечер
- 🧠 Подбор по настроению
- 🎬 Похоже на сериал
- 🎲 Случайный сериал
- 🔥 Популярные подборки
- ❤️ Избранное
- 🚫 Уже смотрел


---

# 4. Флоу "Подобрать сериал на вечер"

Это основной сценарий (MVP).

## Шаг 1 — Сколько времени есть

Сообщение:

```
Сколько времени планируете смотреть?
```

Кнопки:

- 1–2 часа
- 2–4 часа
- Несколько вечеров
- Не важно


---

## Шаг 2 — Формат сериала

Сообщение:

```
Какой формат сериала хотите?
```

Кнопки:

- Мини‑сериал
- 1 сезон
- Несколько сезонов
- Не важно


---

## Шаг 3 — Настроение

Сообщение:

```
Какое настроение сейчас?
```

Кнопки:

- Лёгкое / расслабиться
- Напряжённое
- Смешное
- Атмосферное
- Мрачное
- Романтика
- Удиви


---

## Шаг 4 — Ограничения

Сообщение:

```
Есть ли ограничения?
```

Кнопки‑переключатели:

- Только завершённые
- Без ужасов
- Без тяжёлой драмы
- Рейтинг 7+
- Без российских сериалов


---

## Шаг 5 — Выдача результатов

Бот показывает 3 сериала.


Формат карточки:

```
Название сериала

Год • жанры • рейтинг

Короткое описание

Серий: X
Длительность: X мин
Сезонов: X

Почему подходит:
Короткое объяснение почему сериал подходит под выбранные параметры
```


Кнопки под карточкой:

- ▶️ Смотреть
- ➕ Похожее
- ⏭ Другой вариант
- ❤️ В избранное
- 🚫 Уже смотрел


---

# 5. Флоу "Подбор по настроению"

Сообщение:

```
Что хочется почувствовать?
```

Кнопки:

- 😂 Посмеяться
- 🕵️ Интрига
- 🌌 Атмосфера
- 💔 Драма
- 🤯 Необычный сюжет
- 🛋 Лёгкий фон


После выбора бот уточняет:

```
Какой темп сериала?
```

Кнопки:

- Быстро затягивает
- Спокойный
- Баланс


Далее бот показывает 3 варианта.


---

# 6. Флоу "Похоже на сериал"

Бот спрашивает:

```
Напишите название сериала
```

После ввода:

Бот показывает уточнение:

```
Что именно в нём понравилось?
```

Кнопки:

- Сюжет
- Атмосфера
- Юмор
- Персонажи
- Напряжение


После выбора бот подбирает похожие сериалы.


---

# 7. Флоу "Случайный сериал"

Бот сразу предлагает один сериал.

Карточка как в основном подборе.

Кнопки:

- ▶️ Беру
- 🔄 Другой
- ❤️ В избранное


---

# 8. Популярные подборки

Бот показывает категории:

- Топ сериалы
- Мини‑сериалы
- Лучшие детективы
- Лучшие комедии
- Самые напряжённые


После выбора выводится список карточек.


---

# 9. Избранное

Показывает сохранённые сериалы пользователя.


---

# 10. Уже смотрел

Пользователь может отметить сериал как просмотренный.

Такие сериалы больше не показываются в рекомендациях.


---

# 11. Структура данных сериала

Минимальный набор полей:

```
id
name
original_name
year
rating
votes

poster

is_series
is_mini_series

seasons_total
episodes_total

runtime_episode

status (ended / ongoing)

countries

genres

description

kp_id
imdb_id

binge_score
lightness_score
intensity_score
```


---

# 12. Логика фильтрации

При подборе используются параметры:

- жанр
- рейтинг
- длительность серии
- количество сезонов
- статус сериала
- настроение


Дополнительные веса:

```
binge_score
lightness_score
intensity_score
```


---

# 13. Исключения

Перед выдачей результатов:

Исключить

- сериалы из списка "уже смотрел"
- сериалы с рейтингом ниже заданного


---

# 14. Формирование рекомендаций

Бот должен выдавать:

- 3 основных варианта

Если пользователь нажимает "Другой вариант" — выдаётся новый сериал.


---

# 15. Будущие улучшения

Потенциальные улучшения:

- персональный рейтинг пользователя
- подбор по актёру
- подбор по режиссёру
- подбор по платформе
- рекомендации на основе истории
- рекомендации на основе лайков


---

# 16. Требования к UX

Бот должен:

- использовать inline кнопки
- избегать длинных текстов
- показывать 3 варианта максимум
- позволять быстро сменить рекомендацию


Главная цель — пользователь должен получить сериал для просмотра менее чем за 30 секунд.
